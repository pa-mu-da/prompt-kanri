# Firebase かんたん設定ガイド

Firebaseの操作が初めてでも大丈夫です！以下の手順を一つずつ進めてみてください。

## 1. Firebase プロジェクトの作成
1. [Firebase Console](https://console.firebase.google.com/) にアクセスします。
2. **「プロジェクトを追加」** をクリックします。
3. プロジェクト名（例: `prompt-palette`）を入力し、続行します。
4. Google アナリティクスは「無効」にしても問題ありません。「プロジェクトを作成」をクリックします。

---

## 2. Firestore Database の有効化（データベース）
プロンプトのテキスト情報を保存する場所です。
1. 左メニューの **「ビルド」>「Firestore Database」** を選択します。
2. **「データベースの作成」** をクリックします。
3. **「テストモードで開始」** を選択して「次へ」進みます（後で本番用に修正可能です）。
4. ロケーションはデフォルト（例: `us-central` や `asia-northeast1`）のまま「有効にする」をクリックします。

---

## 3. Storage の設定（スキップしてOK）
今回、画像の保存方法を変更したため、**Storageの設定は不要になりました！**
もしエラーが出て進めない場合は、そのまま飛ばして次の手順に進んでください。

---

## 4. Authentication の有効化（ユーザー識別）
誰がどのデータを編集できるかを分けるための仕組みです。
1. 左メニューの **「ビルド」>「Authentication」** を選択します。
2. **「始める」** をクリックします。
3. **「ログイン メソッド」** タブにある **「匿名」** を探し、スイッチを「有効」にして保存します。

---

## 5. 設定値（APIキーなど）の確認
1. 左上の歯車アイコン ⚙️ > **「プロジェクトの設定」** を開きます。
2. 画面下の方にある「マイアプリ」セクションで、**「< / >」(ウェブ)** アイコンをクリックしてアプリを登録します。
3. 表示される `firebaseConfig` の中身をコピーして、私に教えてください！
   * または、`.env.local` ファイルに貼り付けてください。

> [!TIP]
> **「テストモード」** は30日後に有効期限が切れますが、まずはアプリを動かすためにこれで進めるのが一番簡単です！
